# 🛒 蝦皮退貨自動補庫存工具 (ShopeeReturnSync)

這是一個專為蝦皮賣家開發的 Python 自動化工具。它能自動讀取多份**退貨報表**，並將退貨數量精確地加回至**主庫存模板**（價格及庫存批次更新檔）中，省去手動逐筆對帳補貨的繁瑣流程。

---

## ✨ 主要功能

- **多檔並行處理**：可同時處理 `主庫存/` 資料夾內的所有模板。
- **自動加總退貨**：自動加總多份退貨單中的相同商品/規格數量。
- **智慧識別與保護**：
  - **精準定位**：自動跳過模板頭部的系統標記與說明列（如 `sales_info` 或「必填」說明），只修改真正的商品庫存。
  - **自動修復**：自動處理商品 ID 的格式問題（轉字串、去空格），避免型別不一致導致的錯誤。
- **高相容性引擎**：優先使用 `calamine` 引擎讀取 Excel，比傳統方式更快速且能解決部分 Excel 格式損毀問題。
- **智慧過濾**：自動跳過 Excel 產生的暫存檔（`~$` 開頭）以及已經處理過的檔案（`已補退貨_` 開頭）。
- **完整紀錄**：清楚列出哪些退貨商品在主庫存中找不到（可能已下架），方便人工二次確認。

## 🚀 使用步驟

### 1. 安裝環境
確保已安裝 Python 3，並在終端機執行：
```bash
pip install -r requirements.txt
```

### 2. 準備檔案 (關鍵步驟)
- **主庫存**：放入下載的模板（如果是壓縮檔需先解壓縮）。
- **退貨單**：
  1. 蝦皮導出的退貨單通常有加密。
  2. 請用 Excel 或 Numbers 開啟，輸入密碼（通常為預設密碼）。
  3. **重點**：請點選「檔案」>「另存新檔」，將格式選為 **Excel (.xlsx)** 並確保**不要勾選加密**，放入 `退貨單/` 資料夾。
  > 💡 *小提醒：程式無法處理加密檔案，若讀取失敗請檢查是否已成功另存為不加密格式。*

### 3. 執行程式
在終端機執行：
```bash
python main.py
```

### 4. 上傳更新
程式執行完後，會在 `主庫存/` 資料夾內看到 `已補退貨_` 開頭的檔案。請將這些檔案上傳至：
**蝦皮賣家中心 → 商品管理 → 商品批次工具 → 更新商品資料 → 價格及庫存**

## 🛠️ 技術說明 (為什麼這很重要？)

### 關於「英文欄位名稱」
你在 Excel 看到的是「商品ID」，但程式設定為 `et_title_product_id`。這是因為蝦皮模板的第一列其實是給系統看的英文代碼。本程式直接操作這些代碼，可以**確保上傳到蝦皮時不會因為介面文字變動而失效**。

### 關於「資料列判定」
程式會自動檢查 `et_title_product_id` 是否為「純數字」。
- 如果是 `28905562268` → **判定為資料**，進行補貨。
- 如果是 `sales_info` 或 `商品ID` → **判定為說明列**，自動跳過，確保不破壞檔案結構。

## ❓ 常見問題

- **Q: 為什麼出現 `Value must be one of...` 錯誤？**
  - A: 這是舊版讀取方式 (openpyxl) 的限制。新版已改用 `calamine` 引擎，應可自動解決此問題。
- **Q: 如果我有多份主庫存，會重複計算嗎？**
  - A: 程式會對每一份主庫存檔案進行獨立的比對與補貨。如果同一個商品出現在多份主庫存中，每一份都會被加上該退貨數量（符合蝦皮多店鋪或多檔案管理的邏輯）。
- **Q: 程式會動到我的原始檔嗎？**
  - A: 不會。程式會讀取原始檔，然後另存一個新檔案，你的原始檔會保持原樣。
