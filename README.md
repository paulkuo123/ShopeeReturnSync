# 🛒 蝦皮退貨自動補庫存工具 (ShopeeReturnSync)

這是一個專為蝦皮賣家開發的 Python 自動化工具。它能自動讀取多份**退貨報表**，並將退貨數量精確地加回至**主庫存模板**（價格及庫存批次更新檔）中，省去手動逐筆對帳補貨的繁瑣流程。

---

## ✨ 主要功能

- **Dry-Run 預覽模式**：正式補貨前可先預覽變動紀錄（`dry_run_log.txt`），完全不影響原始檔案。
- **自動解密與過濾**：批次解除報表密碼並自動跳過暫存檔與已處理檔案。
- **安全防護機制**：偵測異常大量退貨與重複檔案，防止庫存數據異常。
- **智慧比對補貨**：自動加總多份報表並精確比對主庫存，確保不重複計算且不破壞模板結構。

## 🚀 使用步驟

### 1. 安裝環境
確保已安裝 Python 3，並在終端機執行：
```bash
pip install -r requirements.txt
```

### 2. 準備檔案 (關鍵步驟)
- **主庫存**：放入下載的模板（如果是壓縮檔需先解壓縮）。
- **退貨單 (解密)**：
  - **方法 A (自動批次解密 - 推薦)**：
    1. 將加密的報表放入 `退貨單未解密/` 資料夾。
    2. 執行 `python3 unlock_excel.py`。
    3. 程式會自動解除密碼並存入 `退貨單/` 資料夾（檔名會加上 `no_pw_` 前綴）。
  - **方法 B (手動另存新檔)**：
    1. 用 Excel 開啟加密報表並輸入密碼。
    2. 點選「另存新檔」，格式選為 **Excel (.xlsx)** 並確保**不要勾選加密**。
    3. 將產生的檔案放入 `退貨單/` 資料夾。
  > 💡 *小提醒：`main.py` 無法直接處理加密檔案，請務必先透過以上任一方法完成解密。*

### 3. 執行程式

**建議流程：先預覽，再正式補貨。**

#### 步驟 3-1：Dry-Run 預覽（強烈建議）
```bash
python3 main.py --dry-run
```
執行後會在終端機顯示：
- 每一筆會被修改的商品（補貨前 → 補貨後）
- 退貨數量超過 20 件的異常警示
- 重複退貨單警告

**此模式不會產生任何輸出檔案**，但會自動在專案根目錄產生一個 `dry_run_log.txt` 文件，紀錄所有終端機輸出的內容，方便紀錄與查閱。確認顯示的變動正確後再執行下一步。

#### 步驟 3-2：正式補貨
```bash
python3 main.py
```

### 4. 上傳更新
程式執行完後，會在 `主庫存/` 資料夾內看到 `已補退貨_` 開頭的檔案。請將這些檔案上傳至：
**蝦皮賣家中心 → 商品管理 → 商品批次工具 → 更新商品資料 → 價格及庫存**

## 🛠️ 技術說明 (為什麼這很重要？)

### 關於「英文欄位名稱」
你在 Excel 看到的是「商品ID」，但程式設定為 `et_title_product_id`。這是因為蝦皮模板的第一列其實是給系統看的英文代碼。本程式直接操作這些代碼，可以**確保上傳到蝦皮時不會因為介面文字變動而失效**。

### 關於「資料列判定」
程式會自動檢查 `et_title_product_id` 是否為「純數字」。
- 如果是 `28905562268` → **判定為資料**，進行補貨。
- 如果是 `sales_info` 或 `商品ID` → **判定為說明列**，自動跳過，確保不破壞檔案結構。

## ❓ 常見問題

- **Q: 為什麼出現 `Value must be one of...` 錯誤？**
  - A: 這是舊版讀取方式 (openpyxl) 的限制。新版已改用 `calamine` 引擎，應可自動解決此問題。
- **Q: 兩個主庫存檔案加起來才是完整賣場，退貨會重複加到兩個檔案嗎？**
  - A: 不會。程式使用 Left Join，以主庫存為主進行比對。某個商品只在 `1.xlsx` 存在，就只有 `1.xlsx` 會更新；`2.xlsx` 完全不受影響。
- **Q: 程式會動到我的原始檔嗎？**
  - A: 不會。程式會讀取原始檔，然後另存一個 `已補退貨_` 開頭的新檔案，原始檔保持原樣。
- **Q: Dry-Run 模式下會產生輸出檔案嗎？**
  - A: 完全不會。`--dry-run` 只在終端機顯示預覽，不寫入任何檔案。
